# 系统性风险 – 第55条

## 判定标准

GPAI模型在以下情况下被归类为具有系统性风险：

- 训练的累计算力超过 **10²⁵ FLOPS**（附件XIII），或者
- 欧盟委员会通过决定对其进行分类

提供者必须在其模型满足该标准后**2周内**通知欧盟委员会。

## 受影响的模型（截至2026年3月）

根据当前认知，被认为或可能被认为具有系统性风险的模型：

- GPT-4/4o (OpenAI)
- Claude 3.5/4 (Anthropic)
- Gemini Ultra/Pro (Google)
- Llama 3.1 405B (Meta)

## BAUER GROUP相关性

**无直接相关性。** BAUER GROUP是这些模型的用户，而非提供者。第55条的义务适用于各模型提供者。

间接相关性：BAUER GROUP应在**供应商选择**（供应商尽职调查）过程中核查GPAI提供者是否履行了其义务。这可以降低将模型集成到客户产品中时的自身责任风险。
