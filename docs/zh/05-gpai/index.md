# 通用人工智能 (GPAI) – 第51–56条

## 定义

**GPAI模型**是可用于多种不同任务的AI模型（例如大型语言模型，如GPT、Claude、Gemini）。它们并非为特定目的而训练，而是可以在训练后适配各种下游任务。

## GPAI模型与GPAI系统的区别

| | GPAI模型 | GPAI系统 |
|---|---|---|
| **是什么** | 经过训练的模型本身 | 集成了GPAI模型的系统 |
| **监管对象** | 模型提供者（OpenAI、Anthropic、Google等） | 系统提供者（将模型集成到产品中的一方） |
| **义务** | 第53条（全部）、第55条（存在系统性风险时） | 取决于系统的风险等级 |

## 系统性风险（第51条）

GPAI模型具有**系统性风险**的条件：

- 训练的累计算力超过 **10²⁵ FLOPS**，或者
- 欧盟委员会据此对该模型进行分类（基于高能力/高影响）

具有系统性风险的GPAI模型提供者承担**额外义务**（第55条）。

## BAUER GROUP视角

BAUER GROUP **不是GPAI模型提供者**。我们不训练自己的基础模型。我们的角色是：

- **GPAI部署者** — 我们将GPAI模型（Claude、GPT等）作为工具使用
- **潜在的系统提供者** — 如果我们将GPAI模型集成到客户产品中

GPAI提供者义务（第53、55条）**不适用于BAUER GROUP**，而适用于Anthropic、OpenAI等。只有当我们以自己的名义将基于GPAI的系统作为高风险系统投放欧盟市场时，责任才会转移。
