# AI治理框架

## 结构

BAUER GROUP将AI治理整合到现有的组织架构中，不设立单独的治理机构。

### 职责

| 角色 | 人员/职能 | AI Act 职责 |
|---|---|---|
| **AI Act 合规负责人** | 管理层 (Karl) | 战略决策、Go/No-Go |
| **技术实施** | 开发负责人 | 需求实施 |
| **文档管理** | 项目管理 | 合规文档维护 |
| **AI Literacy** | 人力资源/入职培训 | 培训实施与记录 |

### 流程

1. **含AI的新产品：** 第5条评估方案 → 风险分类 → Go/No-Go → （如Go）合规实施
2. **现有产品 + AI扩展：** 同新产品流程，但需审查风险等级是否变化
3. **内部AI使用：** 确保AI Literacy，遵守部署者义务
4. **年度审查：** 更新AI清单、审查分类、开展培训

### 文档层级

```
ai-act.docs.bauer-group.com（本站点）
  └── 监管文档（公开）
       ├── 一般合规说明
       ├── 模板和表格
       └── CRA文档参考

内部文档（非公开）
  ├── AI系统清单（特定于产品）
  ├── Go/No-Go 决策记录
  ├── 风险分类（特定于产品）
  └── 培训记录
```
