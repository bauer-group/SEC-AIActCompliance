# 分类决策

## 决策树

针对BAUER GROUP每个已识别的AI系统：

```
1. 它是否属于第3条第1款定义的AI系统？
   │
   ├─ 否 → AI Act 不适用 → 结束
   │
   └─ 是 ↓

2. 它是否属于第5条（禁止的做法）？
   │
   ├─ 是 → 禁止 → 停止开发
   │
   └─ 否 ↓

3. 它是否是 Annex I 产品的安全组件
   且需要第三方合格评定？
   │
   ├─ 是 → 高风险（第6条第1款）→ Go/No-Go
   │
   └─ 否 ↓

4. 它是否属于 Annex III 类别？
   │
   ├─ 否 → 检查第50条（透明度义务）
   │         │
   │         ├─ 是 → 有限风险
   │         └─ 否 → 最低风险 → 结束
   │
   └─ 是 ↓

5. 它是否对自然人进行画像分析？
   │
   ├─ 是 → 高风险（始终）→ Go/No-Go
   │
   └─ 否 ↓

6. 第6条第3款的例外情况是否适用？
   （狭窄的任务范围，不实质性地
    影响人类决策）
   │
   ├─ 是 → 非高风险
   │        → 记录评估（第6条第4款）
   │        → 注册（第49条第2款）
   │
   └─ 否 → 高风险 → Go/No-Go
```

## 实务示例 – BAUER GROUP

| 系统 | AI组件 | Annex III？ | 画像分析？ | 结果 |
|---|---|---|---|---|
| 手写OCR（抄表） | YOLO + TrOCR | 否 | 否 | **最低风险** – 无义务 |
| 客户网站聊天机器人 | LLM集成 | 否 | 否 | **有限风险** – 需标注 |
| AI辅助人员预筛选 | ML排序 | 是（第4a项） | 是 | **高风险** → Go/No-Go |
| 预测性维护（IoT） | 异常检测 | 检查（第2项） | 否 | **需审查** |
| 电子邮件垃圾过滤器 | 分类 | 否 | 否 | **最低风险** – 无义务 |
