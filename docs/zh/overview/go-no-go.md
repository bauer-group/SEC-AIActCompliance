# Go/No-Go 决策框架

## 原则

BAUER GROUP 的每个AI系统在投放欧盟市场之前都要进行**成本效益评估**。AI Act 并未规定此类评估，但作为企业，我们自主决定在何处分销哪些产品。

::: danger 核心规则
**不成比例的监管工作量 = 不在欧盟分销。** 我们没有义务在欧盟提供每一款产品。当欧盟通过监管使创新成本增加时，我们将在没有这些限制的市场分销该产品。
:::

## 决策矩阵

```
┌─────────────────────────────────────────────┐
│ 产品是否包含AI组件？                          │
│                                             │
│   否 → AI Act 不适用 → GO                    │
│   是 ↓                                      │
├─────────────────────────────────────────────┤
│ 系统是否属于第5条（被禁止的做法）？              │
│                                             │
│   是 → 不开发该产品 → STOP                    │
│   否 ↓                                      │
├─────────────────────────────────────────────┤
│ 风险分类（第6条、附件III）                     │
│                                             │
│   最低风险 → GO（无义务）                      │
│   有限风险 → GO（仅透明度义务）                 │
│   高风险 ↓                                   │
├─────────────────────────────────────────────┤
│ 高风险成本效益评估                             │
│                                             │
│   合规工作量 ≤ 3年贡献利润的15%                 │
│   → GO                                      │
│                                             │
│   合规工作量 > 3年贡献利润的15%                 │
│   → NO-GO EU                                │
│   → 评估第三国市场分销                         │
└─────────────────────────────────────────────┘
```

## 高风险合规工作量（估算）

| 项目 | 一次性 | 每年 |
|---|---|---|
| 风险管理系统（第9条） | 40–80小时 | 20–40小时 |
| 数据治理文档（第10条） | 60–120小时 | 10–20小时 |
| 技术文档（第11条、附件IV） | 80–160小时 | 20–40小时 |
| QMS建立与证明（第17条） | 40–80小时 | 20–40小时 |
| 合格评定（第43条） | 40–80小时 | 10–20小时 |
| 欧盟数据库注册（第49条） | 8–16小时 | 4–8小时 |
| 上市后监控 | — | 20–40小时 |
| **高风险总计** | **268–536小时** | **104–208小时** |

按内部时薪120欧元计算，这意味着**一次性32,160–64,320欧元**加上**每年12,480–24,960欧元** — 不包括外部咨询和公告机构费用。

## 阈值计算

示例：包含AI组件的产品，预期3年贡献利润200,000欧元。

- 15%阈值：30,000欧元
- 预估合规工作量（最低）：约57,000欧元（3年）
- **结果：NO-GO EU** — 仅在第三国市场分销

## NO-GO EU 时的替代方案

1. **移除AI组件** — 实施基于规则的替代方案
2. **第三国市场分销** — 亚太、中东北非、拉丁美洲
3. **AI组件作为可选功能** — 欧盟版本不含AI，完整版本用于第三国市场
4. **等待简化** — 等待 Digital Omnibus / 授权法令
