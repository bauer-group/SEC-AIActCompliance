# AI清单与注册

## AI系统清单

每个组织都应维护一份其开发、部署或分销的所有AI系统的清单。这并非AI Act的明确义务，但却是正确进行风险分类和合规评估的前提条件。

### 清单字段

| 字段 | 描述 |
|---|---|
| 系统ID | 唯一标识符 |
| 名称 | 产品名称/内部名称 |
| 描述 | AI功能简要描述 |
| AI组件 | 类型（ML、DL、LLM等） |
| BAUER角色 | 提供者/部署者/分销者 |
| 风险等级 | 禁止/高/有限/最低 |
| Go/No-Go EU | Go / No-Go / 待定 |
| 截止日期 | 相关合规截止日期 |
| 负责人 | 负责人员 |
| 最近审查 | 最近一次分类审查日期 |

模板：[AI系统清单](/zh/templates/ai-inventory)

## EU数据库注册（第49条）

### 注册义务

在投放市场或投入使用之前，**提供者**必须在EU数据库中注册以下系统：

- **高风险AI系统**（第49条第1款）
- **未被归类为高风险的附件III系统**（第49条第2款）

**部署者**如果是公共机构或代表公共机构行事，则必须进行注册。

### 注册内容

所需信息在**附件VIII**中定义：

- A部分：提供者信息
- B部分：高风险AI系统信息
- C部分：部署者信息（仅限公共机构）
